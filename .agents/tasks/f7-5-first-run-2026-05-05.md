## Procedimiento mínimo

### O1.1 — Verificar 5 gates (NO abrir, solo verificar)

Usar comandos desde la VPS. No basta con leer el repo.

```bash
cd ~/umbral-agent-stack

echo "===== repo main/head ====="
git branch --show-current
git rev-parse HEAD
git status --short

echo "===== service state ====="
systemctl --user show umbral-worker.service -p MainPID -p ActiveState -p SubState -p FragmentPath -p DropInPaths
curl -s -o /dev/null -w "HTTP %{http_code}\n" http://127.0.0.1:8088/health

PID=$(systemctl --user show umbral-worker.service -p MainPID --value)

echo "===== L1/L3 env names and values from service process (NO secrets) ====="
tr '\0' '\n' < /proc/$PID/environ \
  | awk -F= '
      $1=="RICK_COPILOT_CLI_ENABLED" {print "process RICK_COPILOT_CLI_ENABLED="$2}
      $1=="RICK_COPILOT_CLI_EXECUTE" {print "process RICK_COPILOT_CLI_EXECUTE="$2}
      $1=="COPILOT_GITHUB_TOKEN" {print "process COPILOT_GITHUB_TOKEN=present_by_name"}
      $1=="WORKER_TOKEN" {print "process WORKER_TOKEN=present_by_name"}
    '

echo "===== L1/L3 envfile values (safe booleans only) ====="
grep -E '^RICK_COPILOT_CLI_(ENABLED|EXECUTE)=' /home/rick/.config/openclaw/copilot-cli.env

echo "===== L2/L4 repo policy ====="
grep -A12 '^copilot_cli:' config/tool_policy.yaml
grep -A8 '^  egress:' config/tool_policy.yaml

echo "===== L5 code gate ====="
grep -n '^_REAL_EXECUTION_IMPLEMENTED' worker/tasks/copilot_cli.py

echo "===== no live egress leftovers ====="
sudo nft list table inet copilot_egress 2>/dev/null && echo "COPILOT_NFT_TABLE_PRESENT" || echo "no copilot nft table"
docker network ls 2>/dev/null | grep -i copilot || echo "no copilot docker network"
```

Reportar en formato:

```text
L1 repo/envfile dice X / VPS process muestra Y
L2 repo dice X / VPS checked-out HEAD muestra Y
L3 repo/envfile dice X / VPS process muestra Y
L4 repo dice X / live nft/docker muestra Y
L5 repo dice X / checked-out code muestra Y
```

### O1.2 — Si los gates permiten run controlado seguro

Solo continuar si:

- L1 `RICK_COPILOT_CLI_ENABLED=true`
- L2 `copilot_cli.enabled=true`
- L3 `RICK_COPILOT_CLI_EXECUTE=true`
- L4 estado de egress permite la ejecucion controlada sin fuga no auditada
- L5 `_REAL_EXECUTION_IMPLEMENTED=True`
- Worker `/health` = 200
- No hay drift repo-vs-process sin explicar

Si cualquiera de esos puntos no se cumple, saltar a O1.3.

Crear scratch:

```bash
TS=$(date -u +%Y%m%dT%H%M%SZ)
SCRATCH="$HOME/umbral-scratch/f7-5-$TS"
mkdir -p "$SCRATCH"
cat > "$SCRATCH/context.txt" <<'EOF'
Este directorio scratch existe solo para F7.5.
Objetivo: generar un README.md de 1 párrafo describiendo este directorio scratch.
EOF
```

Ejecutar exactamente UN run con brief:

```text
Generá un README.md de 1 párrafo describiendo este directorio scratch.
```

Usar `copilot_cli.run` vía Worker API. El input debe usar un repo/workspace
allowlisted o el mecanismo scratch previsto por el handler. Si el handler
rechaza el scratch por allowlist, NO relajar allowlists; reportar verdict
amarillo con la evidencia.

Capturar obligatoriamente:

- `batch_id`
- `agent_id`
- `mission_run_id`
- `tokens`
- `cost_usd`
- `exit_code`
- `duration_sec`
- artefactos generados
- audit JSONL filtrado por `copilot_cli.*`
- journal relevante de `umbral-worker.service` alrededor de la ejecucion

### O1.3 — Si los gates NO permiten ejecutar

No abrir gates. No editar envfiles. No editar `config/tool_policy.yaml`. No
editar `_REAL_EXECUTION_IMPLEMENTED`.

Reportar:

- verdict `amarillo` si la infra esta sana pero falta un gate esperado;
- verdict `rojo` si hay drift peligroso, token leak, worker unhealthy, egress
  sobrante, o evidencia inconsistente;
- que gate bloqueo el run;
- que evidencia se necesita para avanzar.
