## Lecciones operativas

1. **Python no hot-reloadea**: para módulos cargados por daemons long-running (notion-poller-daemon, dispatcher, worker), `git pull` no basta — hace falta restart del proceso. Ya conocido para systemd units; ahora también queda explícito para procesos lanzados por cron wrappers.
2. **Wrappers de cron con `kill -0 $PID && exit 0`** son seguros para evitar duplicados, pero ciegos al staleness del código. Patrón a auditar en otros wrappers del repo (no scope hoy; ver task 2026-05-19-001).
3. **VPS Reality Check Rule confirmada otra vez**: el roadmap decía "daemon respawna naturalmente vía cron"; el runtime decía lo contrario. La verdad estaba en el wrapper, no en el roadmap.
4. **Autorización puntual funciona**: levantar la regla "no pkill" para un único PID, con señal explícita (TERM, no KILL), y con identity-check previo (`ps -o cmd= -p $PID | grep -q notion-poller-daemon`), permitió cerrar el deploy sin abrir un agujero permanente en las reglas.
