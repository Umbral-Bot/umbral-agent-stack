### Implementation

* New module `scripts/discovery/lib/publication_hash.py` (~240 lines):
  * `normalize_publication_text(s) -> str` ŌĆö case-preserving,
    paragraph-aware, whitespace-tolerant. Distinct from
    `lib.dedup.normalize_text` (which lowercases for signal-grade
    dedup).
  * `compute_publication_content_hash(channel, body_text, title="",
    source_content_hash="") -> str` ŌĆö sha256 of `"\n".join([channel.lower(),
    normalize(title), normalize(body), source_content_hash.strip()])`.
  * `ensure_publication_hash_column(db) -> None` ŌĆö idempotent
    `PRAGMA table_info(published_history)` + `ALTER TABLE ADD COLUMN
    publication_content_hash TEXT` + `CREATE INDEX IF NOT EXISTS
    idx_published_history_pub_hash`. Tolerates `sqlite3.OperationalError`.
  * `register_publication_hash(db, content_hash, publication_content_hash)`
    ŌĆö UPDATE on existing `published_history` row.
  * `is_duplicate_publication(db, publication_content_hash) -> bool` ŌĆö
    tolerant of missing column / missing table.
* Extends `scripts/discovery/lib/dedup.py::register_published(...)` with
  optional `publication_content_hash=None` kwarg.
  * When supplied: lazy-imports `ensure_publication_hash_column`, runs
    it, then 5-column `INSERT OR IGNORE`.
  * When `None`: original 4-column INSERT, no migration triggered. **Byte-
    identical to Wave 1.5 path** for back-compat.
