## 3. Resolver — `scripts/copilot_egress_resolver.py`

Stdlib-only Python 3.10+. Read-only on policy and DNS; never mutates
host state.

Behaviour:

- Reads endpoints from `config/tool_policy.yaml`.
- Resolves each via `socket.getaddrinfo(host, port, type=SOCK_STREAM)`.
- Drops loopback, link-local, multicast, unspecified and **private
  (RFC1918)** addresses (a public Copilot endpoint resolving to a
  private address is reported as `private_address_resolved`).
- Splits IPv4 / IPv6, deduplicates, sorts.
- Default output: JSON (machine-readable, schema
  `copilot-egress-resolver/v1`).
- `--format nft`: human-readable diff where every `nft …` invocation
  is rendered **as a comment**, with a banner "DRY-RUN OUTPUT — DO
  NOT PIPE INTO `nft`".
- `--write-cache <path>`: optional cache write. Path **must** resolve
  under `reports/copilot-cli/egress-cache/` or
  `artifacts/copilot-cli/egress-cache/` (both gitignored from F4); any
  other target is refused with `CachePathRefused` and exit 3.
- `--non-strict`: report DNS errors but still exit 0. Default is
  strict (exit 4 on any endpoint error).
- Refuses to run with exit 2 if `copilot_cli.egress.activated == true`.
- Never reads `COPILOT_GITHUB_TOKEN`, `GH_TOKEN`, `GITHUB_TOKEN`, or
  any other env credential. Never spawns a subprocess.

### 3.2 How to interpret the JSON

- `policy_endpoints` — verbatim list from `tool_policy.yaml`. Source of
  truth for what the resolver tried to resolve.
- `endpoints[].v4` / `endpoints[].v6` — addresses returned by the
  system resolver for that host, public only.
- `endpoints[].error` — populated when DNS fails or a private
  address was filtered out.
- `ip_sets.copilot_v4` / `ip_sets.copilot_v6` — deduplicated union
  across all endpoints. **These are the sets that would be written**
  by step 5+, NOT by this script.
- `dry_run: true` and `would_apply: false` are constants — there is
  no flag in this script to flip them.
