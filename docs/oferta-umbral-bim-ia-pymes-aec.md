# Oferta Umbral: Automatizacion BIM + IA para Pymes AEC

## 1. Decision

La senal de mercado observada apunta a una oferta mas fuerte que "IA para AEC"
en abstracto:

- factibilidad mas rapida;
- menos retrabajo entre concepto y BIM;
- automatizacion de tareas repetitivas de modelado y coordinacion;
- traspaso mas limpio hacia Revit o flujos ejecutables.

Para Umbral, eso se traduce mejor en un servicio concreto y vendible:

**Sprint de Factibilidad y Automatizacion BIM con IA**

No se vende "software". Se vende un resultado operativo en un caso real del
cliente.

## 2. Cliente Ideal

### Perfil principal

- estudios de arquitectura pequenos o medianos;
- ingenierias o consultoras BIM con equipos acotados;
- oficinas AECO que ya usan Revit, BIM o Dynamo pero siguen con mucho trabajo
  manual;
- equipos que necesitan iterar rapido sin contratar un equipo VDC grande.

### Senales de dolor

- demasiadas horas no cobradas en pre-diseno o factibilidad;
- muchas iteraciones manuales entre idea, volumetria y modelo BIM;
- coordinacion MEP o documental repetitiva;
- dependencia de una o dos personas "que saben automatizar";
- pipeline comercial reactivo y dificultad para mostrar ROI temprano.

## 3. Propuesta de Valor

**Ayudamos a equipos AECO pequenos o medianos a reducir retrabajo y acelerar
factibilidad, iteracion y tareas BIM repetitivas, dejando un flujo util,
gradual y rentable en un caso real.**

## 4. Promesa Comercial Inicial

Promesa segura para salir a vender sin sobreprometer:

**En 4 semanas dejamos un caso real del cliente mejor resuelto, con una mejora
visible en tiempo, claridad o repetibilidad en una de estas tres capas:**

1. factibilidad temprana;
2. automatizacion BIM disciplinar;
3. traspaso de concepto a modelo ejecutable.

No prometer "10x" ni ahorros universales al inicio. La promesa correcta es:

- menos horas perdidas;
- menos retrabajo;
- mejor velocidad de iteracion;
- y un flujo replicable que el cliente pueda seguir usando.

## 5. Oferta Principal

### Nombre recomendado

**Sprint de Factibilidad y Automatizacion BIM con IA**

### Duracion

- 4 semanas

### Alcance

- 1 caso real del cliente
- 1 flujo prioritario
- 1 equipo pequeno de trabajo

### Tres variantes comerciales

#### Variante A: Factibilidad Express

Para oficinas que necesitan evaluar rapido alternativas de sitio, masa,
capacidad o pre-diseno.

#### Variante B: Automatizacion BIM Disciplinar

Para equipos que quieren reducir tareas repetitivas en Revit, Dynamo, MEP,
documentacion o coordinacion.

#### Variante C: Concepto a BIM sin retrabajo

Para oficinas que quieren ordenar el paso entre analisis temprano, iteracion y
modelo ejecutable.

## 6. Entregables

### Entregables base del sprint

- sesion de arranque y mapa de dolor del flujo actual;
- definicion del caso piloto y criterio de exito;
- diagnostico breve de fricciones, tiempos y retrabajos;
- diseno del flujo objetivo;
- implementacion de una mejora real en el caso elegido;
- demo final con evidencia antes/despues;
- documento de uso operativo;
- backlog priorizado de siguiente fase.

### Entregable visible para el cliente

El entregable debe verse como una mejora tangible, por ejemplo:

- plantilla o flujo de factibilidad repetible;
- automatizacion BIM puntual en Revit/Dynamo;
- pipeline de entrada a modelo mas ordenado;
- paquete de estandares, scripts y criterio de uso;
- mini tablero de seguimiento si aplica.

## 7. Lo que se vende en cada variante

### A. Factibilidad Express

Se vende:

- menos tiempo para evaluar opciones;
- mejor conversacion con cliente o developer;
- mas iteraciones en menos tiempo;
- un marco de decision util para cerrar o descartar oportunidades.

### B. Automatizacion BIM Disciplinar

Se vende:

- ahorro de tiempo en tareas repetitivas;
- menor dependencia de trabajo manual experto;
- mas consistencia en entregables;
- un quick win visible para un champion interno.

### C. Concepto a BIM sin retrabajo

Se vende:

- menos friccion entre idea, analisis y modelo;
- menos rehacer desde cero;
- mayor continuidad entre etapas;
- mas claridad para escalar despues.

## 8. Pricing Inicial

Precios iniciales sugeridos en USD, para validar mercado sin quedar barato ni
sobredimensionado.

### Opcion 1: Diagnostico de entrada

- **USD 900 a 1.500**
- duracion: 1 semana
- objetivo: detectar el flujo con mejor ROI para un sprint

Incluye:

- 2 a 3 entrevistas cortas;
- revision de flujo actual;
- recomendacion priorizada;
- propuesta de sprint.

### Opcion 2: Sprint principal

- **USD 3.500 a 6.500**
- duracion: 4 semanas
- objetivo: resolver un caso real y dejar un flujo replicable

Rango sugerido:

- lado bajo para estudio pequeno con un caso puntual;
- lado medio para ingenieria o consultora con mas actores;
- lado alto cuando hay mas integracion, datos o complejidad BIM.

### Opcion 3: Retainer de continuidad

- **USD 1.200 a 2.800 / mes**
- foco: soporte, mejora incremental, nuevos casos y acompanamiento del champion

Sirve para:

- ampliar el piloto a otra disciplina o equipo;
- mantener backlog de automatizaciones;
- sostener adopcion sin depender de un proyecto grande.

## 9. Oferta Recomendable para salir ya

La mejor puerta de entrada hoy es:

**Diagnostico + Sprint**

Secuencia:

1. diagnostico corto;
2. sprint de 4 semanas;
3. opcion de retainer si el piloto demuestra valor.

Eso calza bien con el modelo propuesto de Umbral:

- servicios para generar caja;

- producto/AIaaS para capturar recurrencia despues.

## 11. Riesgos de Venta y Como Evitarlos

### Riesgo 1: sonar demasiado amplio

Evitar vender:

- "hacemos IA para AEC"

Preferir:

- "resolvemos este flujo concreto en este caso real"

### Riesgo 2: depender de claims vendor

Dos de las senales revisadas vienen de vendors. Sirven como direccion de
mercado, no como prueba neutral absoluta.

Usar esos casos para:

- framing comercial;
- analogias;
- mostrar que el problema existe;

pero no para prometer resultados identicos.

### Riesgo 3: vender automatizacion sin adopcion

El sprint debe dejar:

- responsable interno;
- criterio de uso;
- siguiente backlog;
- y una mejora visible, no solo un script.
