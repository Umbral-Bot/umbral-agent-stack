## Strongest Editorial Opportunity

**Pattern detected**: Multiple sources converge on the tension between AI capability and organizational readiness to adopt it. This applies directly to AEC/BIM:

1. **Andrew Ng** (Batch #349): AI-native teams operate differently — but most AEC teams are not AI-native.
2. **Marc Vidal**: Organizations that forget to innovate decline (Byzantium parallel) + productivity paradox.
3. **Aelion.io (Ivan Gomez)**: Technology only matters if it generates value from day one — the ROI-first AEC mindset.
4. **The B1M**: Data centre construction boom shows where investment flows when there IS organizational readiness.

**Candidate angle**: The gap between AI capability and AEC adoption is not a technology problem — it's an organizational readiness problem. The same tools that are transforming software engineering (Ng) require a different kind of team (governance, process, roles) that most AEC firms haven't built yet.

This connects CAND-001's governance thesis with concrete external evidence.
