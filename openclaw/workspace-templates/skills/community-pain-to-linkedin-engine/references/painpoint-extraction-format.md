# Painpoint Extraction Format

Usar este formato minimo por cada pain detectado:

- `pain`: dolor o friccion principal
- `evidencia`: resumen breve de lo observado
- `lenguaje_literal`: frase exacta o casi exacta del usuario
- `ICP_probable`: tipo de usuario o equipo
- `limitacion`: que no puede resolver hoy
- `necesidad`: que realmente busca
- `severidad`: baja | media | alta
- `confianza`: baja | media | alta

## Regla

Si no hay evidencia suficiente para una columna, dejarla como:

- `no claro`
- `no verificado`

No rellenar huecos con intuicion disfrazada de certeza.
