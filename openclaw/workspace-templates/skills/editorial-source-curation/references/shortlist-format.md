# Shortlist Format

## Output skeleton
Usar este esqueleto por defecto.

### 1. Executive summary
- 3-5 lineas
- decir que temas dominan el lote
- decir que fuentes quedaron fuertes y que huecos existen

### 2. Coverage note
- **time window:**
- **required sources covered:** Gartner, McKinsey, Every, Ruben Substack
- **gaps or inaccessible items:**
- **baseline assumptions:**

### 3. Normalized set
Usar una tabla compacta cuando ayude.

| # | Source | Date | Title | Theme | Access | Score | Confidence | Tag |
|---|---|---|---|---|---|---:|---|---|

### 4. Shortlist
Presentar 3-5 cards rankeadas.

#### 1. [Item title]
- **source:**
- **published:**
- **link:**
- **score:**
- **confidence:**
- **why it fits David:**
- **why now:**
- **angle options:**
- **risks or gaps:**
- **recommended next step:**

Repetir el mismo patron para cada item shortlisted.

### 5. Selection request
Cerrar con una peticion explicita de decision.

Ejemplo:
- `elige #1 para una derivacion directa`
- `elige #2 + #4 para combinar insight + evidencia`
- `pide otra pasada con otra ventana temporal o tema`

## After-selection handoff
Solo despues de la eleccion humana, usar este mini-brief.

### Derivative brief
- **selected item:**
- **core thesis to preserve:**
- **David angle:**
- **audience target:**
- **supporting evidence to keep or verify:**
- **possible hook directions:**
- **desired outcome or CTA:**
- **handoff skill:** `linkedin-david`, `linkedin-content`, or manual drafting
- **automation note:** no asumir publicacion automatica

## Formatting rules
- Mantener el shortlist legible, no exhaustivo.
- Explicar cada score con una razon corta, no con justificacion extensa.
- Hacer visibles ambiguedades y limites de acceso.
- No incluir draft de contenido final antes de la seleccion humana.
