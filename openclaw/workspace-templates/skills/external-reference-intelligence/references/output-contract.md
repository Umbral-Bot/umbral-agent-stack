# Output Contract

## Objetivo

Dar una salida consistente y reusable para chat, handoff o persistencia.

## Campos recomendados

- `reference_type`
- `coverage`
- `evidence_observed`
- `inference`
- `hypotheses`
- `applicable_value`
- `recommended_adaptation`
- `target_project_or_system`
- `persistence_level`
- `traceability_actions`
- `gaps_or_next_step`

## Regla

Si el caso va a terminar en write de alto impacto, usar estos campos como paso intermedio aunque la salida final al usuario sea narrativa.
