# Traceability Checklist

## Checklist minimo antes de decir "cerrado"

- [ ] la referencia fue observada con evidencia real o material provisto suficiente
- [ ] el tipo de referencia quedo clasificado
- [ ] evidencia, inferencia e hipotesis quedaron separadas
- [ ] el valor aplicable quedo explicitado
- [ ] se decidio donde encaja o por que no encaja
- [ ] se reviso si ya existia un artefacto o entregable similar
- [ ] se evito crear una pagina suelta fuera del flujo estructurado

## Si el caso merece persistencia

Confirmar ademas:

- [ ] proyecto identificado en `📁 Proyectos — Umbral`
- [ ] artefacto util creado o actualizado en la carpeta compartida
- [ ] entregable creado en `📬 Entregables Rick — Revision` si hacia falta revision
- [ ] seguimiento creado o actualizado en `🗂 Tareas — Umbral Agent Stack` o Linear si se activo trabajo
- [ ] el cierre dice exactamente que se hizo y donde quedo

## Regla de deduplicacion

Antes de crear algo nuevo, revisar si ya existe:
- analisis similar del mismo referente
- entregable pendiente de revision sobre el mismo tema
- nota o artefacto ya alojado en la carpeta compartida
- tarea o experimento ya abierto

Si ya existe una pieza equivalente, actualizarla o enlazarla.
