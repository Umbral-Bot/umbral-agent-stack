## Narrativa profesional — Arcos de historia

### Arco 1: El arquitecto que programo por necesidad

```
No estudie programacion. Estudie arquitectura.

Pero en 2017, coordinando proyectos BIM, me di cuenta de que el 60% de mi
tiempo era tareas que podrian hacerse automaticamente. Aprendi Dynamo.
Despues Python. Despues la API de Revit.

No para ser programador. Para dejar de hacer lo que una maquina podia hacer
mejor.

Eso me llevo a OXXO Chile, a mas de 80 proyectos anuales, a una reduccion
del 40% en costos. Y a ensenarselo a otros arquitectos.
```

### Arco 2: El puente entre mundos

```
Hay arquitectos que saben de BIM pero no de codigo.
Hay programadores que saben de codigo pero no de construccion.
David esta en el medio.

Entiende lo que el coordinador BIM necesita porque lo fue durante anos.
Entiende las limitaciones del modelo Revit porque las vive diariamente.
Y sabe como automatizarlas porque lo hizo, con metricas reales.
```

### Arco 3: La docencia como multiplicador

```
Cuando automatice mi primer flujo BIM, lo hice para mi.
Cuando lo ensene la primera vez, lo hice para 15 personas.
Hoy, los alumnos del Master AEC 4.0 lo hacen desde el primer modulo.

El Citizen Developer no es una tecnologia. Es un cambio de mentalidad:
tu no necesitas saber programar para resolver tus propios problemas con codigo.
Necesitas entender el problema lo suficientemente bien.
```

## Tono de voz — Guia de escritura

### Caracteristicas del tono

- **Directo:** Sin introduccion innecesaria. El primer parrafo ya es el punto.
- **Tecnico pero traducido:** Usa terminos correctos del sector. Los explica cuando es necesario.
- **Basado en datos propios:** Cada afirmacion tiene un numero o un caso real detras.
- **Sin arrogancia:** Los logros se mencionan como contexto, no como autopromocion.
- **Humor discreto:** Ocasionalmente, siempre con base tecnica.

### Lo que David diria

- "Les armo un piloto en 3 semanas. Si no ven resultados, no seguimos."
- "Esto que hacen en Excel les consume mas de la mitad de las horas."
- "Hice exactamente esto para OXXO Chile. Los costos bajaron 40%."
- "No necesitas saber C#. Python + Dynamo resuelve el 80% de los casos."

### Lo que David NO diria

- "Soy apasionado por la transformacion digital." (buzzword vacio)
- "Nuestra vision es un mundo mas conectado." (narrativa corporativa)
- "Con mas de una decada de solidas trayectoria..." (curriculo estilo HR)
- "Estoy entusiasmado de compartir este logro." (falsa modestia corporativa)

## Checklist de marca personal

### Para cualquier bio o presentacion

- [ ] Incluye al menos una metrica concreta (-40%, 90 segundos, +80 proyectos)
- [ ] Menciona la interseccion BIM + Automatizacion + IA (no solo uno de los tres)
- [ ] Tiene contexto LATAM / espanol (diferenciador geografico)
- [ ] Menciona la certificacion ACI o buildingSMART si es contexto tecnico
- [ ] El tono es directo, no corporativo
- [ ] Sin buzzwords de la lista negra
- [ ] Adaptado a la audiencia especifica del contexto
