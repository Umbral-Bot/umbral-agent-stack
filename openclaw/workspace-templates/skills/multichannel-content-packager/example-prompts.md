# Example Prompts

## Example 1

User request:

"Take this approved master draft about BIM coordination errors and package it for LinkedIn, X, blog, and newsletter. Keep the thesis intact. Mark anything that still needs review."

Expected behavior:
- preserve the original claim and proof points
- create one canonical master version
- produce four channel-native versions
- keep X concise
- expand blog structure materially
- return a status matrix
- end with an approval protection note

## Example 2

User request:

"Turn this approved launch note into LinkedIn, X, newsletter, and future Instagram. Do not make Instagram fully ready if visuals are still undefined."

Expected behavior:
- draft Instagram as concept plus caption plus visual note
- mark Instagram `needs review` if asset direction is missing
- avoid cloning the same hook across channels

## Example 3

User request:

"Make an organic LinkedIn post and a paid LinkedIn variant from the same approved message."

Expected behavior:
- keep both versions aligned on thesis
- make the organic version more conversational and insight-led
- make the paid variant tighter and more value-first
- avoid assuming ad setup or publishing actions

## Example 4

User request:

"This is for a LinkedIn newsletter edition, not an email newsletter. Package it alongside X and blog."

Expected behavior:
- switch newsletter treatment to article-style LinkedIn format
- avoid subject line and preview text unless the user also wants an email version
- keep X and blog derived from the same canonical master

## Example 5

User request:

"We have X Premium, so make one long-form X post instead of a thread."

Expected behavior:
- explicitly use the longer-post allowance because the user confirmed it
- still keep the copy concise and front-loaded
- do not assume the same allowance in future requests unless the user confirms it again
