## Lessons learned

1. **Repo vs VPS discipline confirmed.** The 5A PR (`0d6ad83`) exists on remote
   branch but is not in `main` HEAD `54720d7`. The live worker code (not
   restarted since `1418206`) still executes with the old main. Until the PR
   merges AND the worker restarts (picking up new code from disk), L5 stays
   `False` on both the live code path and the checked-out main.

2. **Three-gate protection working as designed.** L3 (operator-controlled envfile),
   L4 (no live nft table provisioned), and L5 (code constant) are each
   independent blockers. Even if one were accidentally flipped, the other two
   would prevent real execution.

3. **COPILOT_GITHUB_TOKEN in process env.** The token is present by name (loaded
   from `copilot-cli-secrets.env` via EnvironmentFile drop-in). This means the
   plumbing is ready — when the operator explicitly opens L3, the token will be
   available to the Docker sandbox without any additional setup.

4. **Egress IP sets are empty.** The nft template was validated in 4C but the
   resolver has not populated `copilot_v4`/`copilot_v6` sets. Before F8,
   the resolver must run and populate the sets, then `nft -f` apply must happen.
